- список аккаунтов

### Что нужно сделать

1. **Исправить ошибку данных**
- Найди причину ошибки в блоке списка аккаунтов.
- Добавь безопасную нормализацию данных перед рендером.
- Если элемент не является словарем аккаунта, не ломай страницу.
- Используй защиту через `isinstance(acc, dict)` и fallback-значения.
- Если данные повреждены, показывай дружелюбное сообщение в UI, а не traceback.

2. **Сделать полноценный empty state**
- Если аккаунтов нет, вместо ошибки или пустого блока покажи красивое состояние:
  - заголовок
  - короткое описание
  - кнопку “Добавить первый аккаунт”
  - пояснение, какие платформы поддерживаются

3. **Улучшить форму добавления аккаунта**
- Сделай форму логичнее и современнее.
- Поля:
  - Платформа
  - Username
  - Способ авторизации
  - Cookies / API Key / Session data
  - Примечание
  - Теги
- Показывай поля динамически в зависимости от выбранной платформы и способа авторизации.
- Добавь валидацию:
  - username обязателен
  - защита от дублей
  - проверка пустых значений
  - проверка JSON для cookies, если выбран такой формат
- После успешного добавления показывай success message.

4. **Переделать список аккаунтов**
- Вместо сырого вывода сделай удобный список или таблицу.
- Для каждого аккаунта отобрази:
  - ID
  - Платформу
  - Username
  - Статус
  - Метод авторизации
  - Дату создания
  - Последнюю проверку
  - Теги
- Добавь действия:
  - Редактировать
  - Обновить сессию
  - Проверить
  - Удалить
- Добавь confirm перед удалением.

5. **Добавить фильтры и поиск**
- Поиск по username / ID
- Фильтр по платформе
- Фильтр по статусу
- Фильтр по способу авторизации

6. **Улучшить KPI блок**

  - Ошибки авторизации
  - Требуют переподключения

7. **Улучшить визуальный UX/UI**
- Сделай страницу более чистой, профессиональной и похожей на современную internal admin panel.
- Убери технический шум.
- Усиль визуальную иерархию:
  - header страницы
  - блок KPI
  - блок фильтров
  - блок формы
  - блок списка аккаунтов
- Используй аккуратные контейнеры, карточки, отступы, статусы и цветовые бейджи.
- Интерфейс должен хорошо работать на desktop и laptop.

8. **Добавить статусы аккаунтов**
Используй статусы:
- active
- warning
- expired
- banned
- draft

Для каждого статуса сделай цветной badge.

9. **Добавить демо-данные**

10. **Сделать код чистым**

- Не ломай существующую структуру проекта.
